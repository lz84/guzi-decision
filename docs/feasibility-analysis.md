## 二、从 Polymarket 获胜视角的需求分析

### 2.1 Polymarket 获胜核心逻辑

Polymarket 是一个预测市场平台，用户通过押注事件结果获利。要在 Polymarket 上获胜，需要：

```
信息优势 + 分析能力 + 执行速度 = 获胜概率
```

**获胜三要素**：

| 要素 | 说明 | 舆情系统支撑 |
|------|------|-------------|
| **信息优势** | 比市场更早获取关键信息 | 实时数据采集、全网覆盖 |
| **分析能力** | 正确解读信息影响 | 情感分析、事件提取、影响评估 |
| **执行速度** | 快速反应市场变化 | 自动化预警、API对接 |

### 2.2 Polymarket 相关舆情需求

#### 需求一：政治事件监测

| 事件类型 | 数据源 | 关键指标 | 预警阈值 |
|----------|--------|----------|----------|
| 选举动态 | Twitter、新闻、民调 | 候选人提及量、情感变化 | 24h情感变化 > 20% |
| 政策变化 | 政府网站、新闻 | 政策关键词出现频率 | 首次出现关键词 |
| 国际关系 | 国际新闻、外交公告 | 地缘政治事件 | 突发事件检测 |
| 立法进程 | 国会公告、新闻 | 法案进展追踪 | 关键节点提醒 |

#### 需求二：经济事件监测

| 事件类型 | 数据源 | 关键指标 | 用途 |
|----------|--------|----------|------|
| 央行政策 | 央行公告、讲话 | 利率预期、通胀预期 | 经济预测市场 |
| 经济数据 | 统计局发布 | GDP、CPI、就业数据 | 宏观事件押注 |
| 企业财报 | 公告、新闻 | 业绩预期vs实际 | 公司相关市场 |
| 大宗商品 | 商品新闻、库存数据 | 供需变化 | 商品价格预测 |

#### 需求三：社会事件监测

| 事件类型 | 数据源 | 关键指标 | 用途 |
|----------|--------|----------|------|
| 公共卫生 | WHO、CDC、新闻 | 疫情数据、政策变化 | 公共卫生市场 |
| 自然灾害 | 气象局、新闻 | 灾害预警、影响评估 | 灾害相关市场 |
| 社会运动 | 社交媒体、新闻 | 话题热度、参与规模 | 社会事件市场 |
| 科技突破 | 科技媒体、论文 | 新技术发布、突破新闻 | 科技预测市场 |

### 2.3 舆情信号转化为预测优势

```
舆情数据 → 信号提取 → 概率调整 → 押注决策

示例流程：
1. 采集：监测到某候选人负面新闻激增
2. 分析：情感分析判断为重大负面（-0.8）
3. 评估：历史类似事件导致支持率下降 3-5%
4. 决策：调整预测概率，考虑押注对手
```

