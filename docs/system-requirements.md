## 一、系统概述

### 1.1 项目背景

谷子情报分析系统是一个面向 Polymarket 预测市场的舆情情报收集与分析平台。系统通过多源数据采集、情感分析、事件提取等技术，为用户提供实时舆情信号，帮助用户在预测市场中获得信息优势。

### 1.2 系统定位

```
舆情数据采集 → 情感分析 → 事件提取 → 预警推送 → 决策支持
```

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 信息优势 | 比市场更早获取关键信息 |
| 分析能力 | 正确解读信息影响 |
| 执行速度 | 快速反应市场变化 |

## 三、舆情处置与预警场景设计

### 3.1 舆情处置闭环流程

```
舆情发现 → 分级评估 → 预警推送 → 处置跟踪 → 复盘分析
     │           │           │           │           │
     ▼           ▼           ▼           ▼           ▼
 自动检测     规则引擎     多渠道推送   状态跟踪    效果评估
```

### 3.2 分级预警规则

| 预警级别 | 触发条件 | 推送渠道 | 响应时限 |
|----------|----------|----------|----------|
| **红色** | 重大负面事件、情感分数 < -0.8、影响力 > 0.9 | 飞书+短信+Webhook | 立即 |
| **橙色** | 中度负面事件、情感分数 < -0.5、影响力 > 0.7 | 飞书+Webhook | 5分钟内 |
| **黄色** | 轻度负面事件、情感分数 < -0.3、影响力 > 0.5 | 飞书 | 15分钟内 |
| **蓝色** | 信息提醒、关键词命中、新话题出现 | 日报 | 次日 |

### 3.3 预警处置跟踪

| 状态 | 说明 | 后续动作 |
|------|------|----------|
| 待处理 | 预警已生成，未确认 | 自动提醒升级 |
| 处理中 | 已确认，正在处置 | 定时状态更新 |
| 已解决 | 处置完成 | 记录处置结果 |
| 已归档 | 复盘完成 | 纳入知识库 |

### 3.4 溯源分析场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 事件溯源 | 事件ID | 事件起源、传播路径、关键节点 |
| 话题溯源 | 关键词 | 首次出现时间、来源平台、传播趋势 |
| 实体溯源 | 实体名称 | 相关事件、情感变化、影响评估 |
